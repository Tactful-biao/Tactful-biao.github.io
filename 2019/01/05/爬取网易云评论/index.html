爬取网易云评论

发表于 2019-01-05 | 分类于 Python | | 浏览 次
字数统计: 1,343 | 阅读时长 ≈ 5

思路

  • 找到评论源地址
  • 构造请求

整体思路就是这些，下面就是围绕这两个大思路进行的。

找评论源地址

这一步是很简单的，打开网页版网易云音乐的详情页，在Chrome浏览器–>F12–>Network。刷新网页，仔细找一下就能找到下图所示的链接，这里面就包含着我们需要的评论内容。其中commonts是评论，hotComments是热评，我们暂时只获取热评。

构造请求

发现目标之后，就可以进行模拟获取内容了，但是网易云音乐毕竟是大厂，如果就这样简简单单的就被别人拿了数据，那多没面子，所以它加了加密的参数。如果想要拿到它的数据，就必须知道这两个加密参数是怎么来的。如下图：

我们需要搞明白它的加密方式，才能正确的模拟它的访问，一般加密方法都是写在js里面的。于是我们就从js里去寻找他的加密方式。

加密参数的探索

在core开头的js里我们通过把js复制到支持搜索的文本编辑器里面，通过查找encSecKey的方式搜索到相关的代码如下：

可以看到两个加密参数都在这里面。分别是bMi1x.encText，另一个是bMi1x.encSecKey。现在问题转移到bMi1x上面来了，bMi1x是通过asrsea加密后得到的，我们暂时先不管asrsea是怎么加密的，我们先把里面的四个参数搞清楚。

  • 第一个参数：json格式的 i7b
  • 第二个参数：bvL5Q([“流泪”, “强”]
  • 第三个参数：bvL5Q(VK4O.md)
  • 第四个参数：bvL5Q([“爱心”, “女孩”, “惊恐”, “大笑”])

其中第二和第四个参数是固定的，即每次都不会变，我们需要注意的是第一个和第三个参数是不是变化的，我们通过查找VK4O.md发现，它也是一个固定的值

Fiddler4 的使用

那么我们就需要判断i7b到底是什么了。下面说一个简单的方法，通过Fiddler4，可以在线修改js进行调试。

首先找到core开头的js，右键Save as，保存到本地core.js就像如下这样：


然后通过设置系统代理是流量通过Fiddler，然后重新刷新刚刚那个网易云链接，可以看到许多的链接，找到那个core开头的js(如果没找到，是因为浏览器缓存的问题。把浏览器关掉，重新打开)，点击右边的AutoResponder，将那个js拉过来，然后点击Rule Editor下面那个Find a file，用本地之前保存的那个core.js替换，然后Save就可以了。如下图
